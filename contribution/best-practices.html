Bugs

General Info

Bugs should contain a bit of information so we or someone else can resolve it properly.
If you would like to report a bug with one of our projects, please ensure you choose the right one.
This means if you had an issue with our Database API, Website or Scanning API please use the appropriate GitHub to report the issue.
This is because we use different Programming languages and systems of handling issues internally once they are reported depending on programming language.

Improvements

General Info

If you believe you found something that could be done / handled in a better way or think you might have a feature / library that interests us we would like to listen.
Improvements should be held relatively short and should just talk about how the improvement might make our project better

Format of Improvement Reports

Same as Bug reports, improvements also have a certain format we want them to follow.

The format is:

  • Improvement: A short description of the improvement you think you have (1-2 Sentences)
  • Implementation Details: How you'd like to see the improvement implemented into our existing software stack
  • Optional: Libraries or other things that may make this improvement possible / easier to do

Documentation

General Info

If you believe you found some way to improve our documentation, or have trouble understanding a piece of code or documentation this is what you want to create!

Format of Documentation Reports

Like everything else, we also want documentation reports to follow a certain format

The format is:

  • Documentation Issue: A short sumup on what you are having trouble understanding (line of code or documentation link and line)
  • Improvement Details: How you'd improve the documentation into something more readable (if possible)
  • Optional: Additional help pages that might help us improve the documentation or fix errors with it
