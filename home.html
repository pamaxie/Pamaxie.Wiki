About Pamaxie

Pamaxie is a publicly accessible content filter aimed at software and hardware developers. 
Utilizing it, they can determine what type of data was sent through their system and if it is safe for their end users. 

The Problems we are tackling

We plan on developing on an open source, self-hostable API that allows its users to scan media send to it. 
We plan on detecting malicious files, hate speech, false information and adult content through it. 
This should hopefully make it easier for website and application developers to focus on their main business instead of developing their own content detection systems. 
We hope this saves developer time, effort and money implementing systems of the same scale as ours.

How we plan to solve them

We plan on solving the problems through open research in the various fields required, working together with other specialists to find a solution to our problems. 
For image detection and filtering we decided on using machine learning implementations to detect various tags for images. 
This includes for example Nazi symbolism and nudity. After that we deliver the results with the media type we detected. 
Media type detection is done through magic numbers, a few bytes at the start of the file’s binary data. 
This is a method to detect the media type based on the binary data of a format. 
This works because many formats start their data with a number of bytes to identify it. 

We are implementing Video detection, which is going to be supported in the future. 
It is using a combination of iFrame extraction and frame deltas to get keyframes from videos via FFMPEG. 
After this is completed, the extracted frames are ran through a neural network. 
Then, like the other solutions we return the media type and detected labels for the data. 
What is slightly different with Video detection is that we also deliver the rough timestamps which contained certain labels (for example Nazi symbolism). 

Identification of different types of known exploits is also planned, but the implementation is still being discussed. 
This would include detecting known viruses or exploits like zip bombs or ransomware. 
We are probably only going to work over a database of banned software hashes to keep the solution maintainable for us. 
We are also looking into solutions involving AV software. This solution will not be developed on for a while. 

Text classification is also planned to allow identification of hate speech or the spreading of wrong information online. 
This solution is still being discussed and also probably not going to be worked on for a while. Its implementation would work based on natural language processing. 
We will also host an instance of the API making it even more accessible for smaller companies/projects who don’t have the resources to host it themselves. 
This will work over a registration of an account and an application on our website. 
This website will also serve as a central hub for companies, or us to manage the API controllers from.
